DIE IDE IN EINEM SATZ

Keine Blackbox.
Du siehst jeden Schritt.

⌘

Live im Editor

Der Agent schreibt und ändert Code direkt dort, wo du ihn siehst. Du kannst jederzeit stoppen, eingreifen oder die Richtung ändern.

▸

Live-Konsole

Syntaxchecks, Tests, Linter und Builds erscheinen mit vollständiger Ausgabe im sichtbaren Terminal.

◇

Mehr als eine Datei

Dateibaum, Ordner, Tabs, Dirty-State und Syntaxdarstellung gehören dazu. Der Agent prüft Mehrdatei-Änderungen vollständig.

01 / SCHNELLSTART

In zwei Befehlen
zur laufenden IDE.

01

Node.js bereithalten

Du brauchst Node.js inklusive npm. Für den Codex-Modus kommt zusätzlich die offizielle Codex CLI dazu.

02

Installieren & starten

npm install -g codex-ide
codex-ide start

Danach öffnet sich die IDE unter http://localhost:8787/.

03

Projekt loslegen

Dein sicherer Arbeitsbereich liegt immer in model-home/. Erstelle Dateien oder gib dem Agenten die erste Aufgabe.

CODEX AUTH-BRIDGE

Dein Codex-Login
bleibt deiner.

Die IDE speichert und kopiert keine Tokens. Sie nutzt die offizielle CLI-Anmeldung und zeigt Status und verfügbare Limits direkt in den Einstellungen.

Für Termux, SSH oder Headless-Setups:

codex-ide codex login --device-auth
codex-ide codex statusPrüft Anmeldung und Gateway-Status.

03 / SKILL-SYSTEM

Mach die IDE
zu deiner IDE.

Unter Einstellungen → Skills richtest du integrierte und installierte Skills für Local, API und Codex ein.

codex-ide skill validate skill_packages/PDF_Skill
codex-ide skill pack skill_packages/PDF_Skill

Externe Einstiegspunkte führen JavaScript aus und brauchen deshalb deine ausdrückliche Vertrauensbestätigung.

04 / SICHERHEIT

Leistung braucht
klare Grenzen.

01

Workspace geschützt

Aktionen bleiben auf model-home/ begrenzt. Absolute Pfade, Traversal und ausbrechende Symlinks werden blockiert.

02

Lokales Gateway

Das Gateway bindet standardmäßig nur an Loopback und blockiert fremde Browser-Hostnamen gegen DNS-Rebinding.

03

Isolierte Vorschau

HTML-Vorschauen bleiben read-only im Workspace und laufen in einer CSP-Sandbox.

04

Sichtbarer Editor

Änderungen landen ausschließlich im sichtbaren Editor, nicht unsichtbar irgendwo im System.
